Four parallel experiments · 2025–2026

Four experiments in making AI visible.

A React Flow mission control, a Sankey diagram of the AI value chain, a bioluminescent 3D brain, and the monorepo they all fed into. Each one answering a different question about the same system.

4 experiments Vite · React · Three.js · D3 Single-operator workbench
Premise

Same system, four angles.

"If I'm going to rely on an AI operating system every day, I need to see it — how it's built, what it costs, how it thinks, and what ships."

AiOS isn't a single product. It's the name for a stack of files, skills, agents and models that sits on top of Claude Code and shapes how one person works. Over a few months, four visualizations emerged — not as a roadmap, but as answers to questions that kept coming up.

Each experiment is a Vite project that runs locally, under a hundred lines of config, with no external dependencies beyond the libraries it needs to do its one job.

The arc

From mapping to shipping.

The four experiments aren't versions of each other. They're a sequence of different questions — and the answers build on each other.

AiOS 1 · Experiment 01
A mission control for an AI operating system.
React Flow WebSocket Express
▸ What does my AI setup actually look like — and how does it map to me?

A React Flow graph with two node sets rendered in the same canvas: the AI system (Model, Memory, Perception, Reasoning, Skills, Orchestration) and a neuroscience model of the brain (Prefrontal, Parietal, Temporal, Amygdala…). Edges between them encode the analog: memory is hippocampus, skills are cerebellum, the LLM is cortex.

An Express + WebSocket backend reads the operator's local filesystem — memory files, skill definitions, sessions, CLAUDE.md configs — and streams real events into the graph. A toggle overlays ADHD severity maps on the brain side, so the visualization doubles as a self-knowledge tool.

AiOS 2 · Experiment 02
A Sankey of what AI actually costs.
D3 Sankey Vite Public talks
▸ Where does a prompt really begin — and who pays for it?

A D3 Sankey that traces the AI value chain from natural resources — energy, minerals, fresh water — through fabrication, data centers, foundation models, API layers, agent frameworks, and finally a single end user running a prompt. Every flow is cited inline, so the artifact doubles as a sourced argument.

Built for non-technical audiences at Bæredygtig Herning and public workshops. The point isn't to guilt anyone — it's to make the invisible stack visible long enough that people stop treating "the AI" as a frictionless service.

llm-brain · Experiment 03
A bioluminescent brain you can orbit.
Three.js Blender 8k neurons
▸ How do you show a neural network to someone who's never seen one?

A Three.js scene rendering 8,000 neurons and 47,167 connections distributed across eight anatomical regions — Prefrontal Cortex, Motor Cortex, Parietal, Temporal, Occipital, Cerebellum, Brain Stem, Amygdala — each color-coded. Activation pulses travel along pre-defined circuits (visual processing, motor planning, memory recall).

A blender-import.py pipeline ports the same geometry into Blender for high-resolution renders used in printed material. The browser version falls back to a 2D projection on machines without a GPU — because half the audience is presenting from a ten-year-old laptop.

The payoff is the handshake: "an LLM is a brain-shape, just made of math instead of tissue." Not literally true. Useful anyway.

AiOS 3 · Experiment 04
The monorepo where the experiments become product.
Monorepo Supabase ai-core
▸ What survives when you commit to shipping it?

AiOS 3 is the consolidated home for production-bound work. A strict monorepo split: apps for interfaces (web, admin), services for the API and integrations, and packages for the shared primitives — UI, types, and an ai-core that owns every prompt and model-routing decision.

One rule matters more than the others: packages never import from apps. The dependency graph points one way. This is the lesson carried forward from AiOS 1 — when every piece can reach every other piece, the system stops being legible, and you're back to needing a React Flow dashboard just to find where a prompt lives.

apps/ ├── web/ // end-user interface └── admin/ // backoffice services/ └── api/ // backend + integrations packages/ // ← one-way imports only ├── ui/ // shared components ├── ai-core/ // prompts, routing, tools └── shared-types/ // contracts

What carried over

Four small projects, four ideas that stuck.

01
Visualize before you ship.
AiOS 1 kept revealing connections that looked fine in code and wrong on a canvas. If a system's mental model doesn't survive being drawn, the model is wrong.
02
Cost is a first-class view.
AiOS 2 made "just one more agent" feel different after watching energy and water flow into the same diagram. The Sankey stays — even inside technical reviews.
03
Metaphor is a tool.
llm-brain isn't accurate neuroscience. It is an honest handle for people who'd otherwise check out. A wrong-but-useful picture beats a correct-but-ignored one.
04
One-way imports.
AiOS 3's strictest rule — packages never reach into apps — is the simplest thing that kept the map from collapsing again. Most architecture is just direction.
